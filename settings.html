How to get these:
  1. Go to console.cloud.google.com and create a project (e.g. "Mama Hub")
  2. APIs & Services → Library → enable Google Drive API and Google Calendar API
  3. APIs & Services → Credentials → create an API key and an OAuth 2.0 Client ID (Web application)
  4. Under the OAuth client's "Authorised JavaScript origins", add your hub's URL (e.g. https://hub.mama.agency) — this is why the hub must be hosted first
  5. Paste both values above and save

Path to full integration (cloud storage & sync)
1
Host the hub on your domain
Push these HTML files to GitHub Pages or Netlify and point your domain (e.g. hub.mama.agency) at it. This unlocks Google OAuth and makes the hub reachable by the whole team.
2
Create the Google Cloud project & paste credentials above
Follow the steps in the Google API card. Once the OAuth origin matches your hosted URL, the hub can talk to Google on each person's behalf.
3
Switch storage from browser to Google
Every tool reads/writes through one small storage layer. Swapping that layer to Google Drive (a JSON file per store) or Google Sheets gives real multi-device, multi-person sync — briefs export directly into client folders, meetings pull from calendars, and timesheets become company-wide. No tool needs rebuilding.
4
Replace PINs with Google sign-in
Once hosted with OAuth, each person signs in with their Mama Google account. That gives true per-person privacy (to-dos, timesheets) and role-based access (only the COO opens Admin) — replacing the current PIN/password deterrents, which are not real security.
